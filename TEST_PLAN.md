# TEST_PLAN.md — MBTI Analyzer 검증 계획

검증은 아래 4단계 순서로 진행한다.

| 단계 | 누가 | 언제 | 목적 |
|---|---|---|---|
| 1단계 자기검토 | AI | 1회 | 코드 로직 오류 발견 |
| 2단계 pytest | 컴퓨터 | 코드 변경 시마다 | 백엔드 API 자동 검증 |
| 3단계 체크리스트 | 사람 | 1회 | 자동화 불가 항목 눈으로 확인 |
| 4단계 Playwright | 컴퓨터 | 코드 변경 시마다 | 브라우저 E2E 흐름 자동 검증 |

### 테스트 수정 금지 원칙

> 테스트가 실패했을 때 테스트 코드를 고쳐서 통과시키지 않는다.
> 반드시 구현 코드를 수정해서 테스트를 통과시킨다.

- 테스트는 기대 동작의 기준이다. 기준을 바꾸는 게 아니라 구현을 기준에 맞춘다.
- 테스트를 수정해도 되는 경우는 **요구사항 자체가 바뀌었을 때**만이다.

## 1단계 — 자기검토 (코드 레벨)

서브 에이전트가 코드를 직접 읽고 아래 항목을 검토한다.

### 1-1. 원래 목표 대비 누락 확인
- [ ] 카카오톡 iOS .txt 파싱이 정상 동작하는가
- [ ] 화자별 메시지 분리가 되는가
- [ ] 날짜 범위 선택 후 해당 범위만 LLM에 전달되는가
- [ ] MBTI 4글자 코드가 결과에 정상 포함되는가
- [ ] 마크다운 보고서가 형식에 맞게 출력되는가

### 1-2. 논리 오류 확인
- [ ] 축별 분석 결과(E/I, S/N, T/F, J/P)와 헤더 MBTI 코드가 일치하는가
- [ ] 타겟 화자 발화만 근거로 사용하는가 (다른 화자 혼입 없는가)
- [ ] 날짜 필터가 date_from ≤ date ≤ date_to 범위를 정확히 자르는가
- [ ] 6000자 한도 초과 시 최근 메시지 우선으로 잘리는가
- [ ] 20개 미만 메시지 시 400 에러가 정상 반환되는가
- [ ] SSE 스트림이 stage → token → done 순서로 오는가

### 1-3. UX 문제 확인
- [ ] 에러 발생 시 JSON 원문이 아닌 한국어 메시지가 표시되는가
- [ ] 날짜 선택 범위 밖 선택 시 경고 문구가 보이는가
- [ ] 분석 중 스트리밍 텍스트가 실시간으로 보이는가
- [ ] 결과 화면에서 공유/다운로드가 이미지로 동작하는가

## 2단계 — 백엔드 pytest (API 자동화)

`backend/tests/test_api.py` 를 작성하고 실행한다.

### 2-1. /api/health
| 케이스 | 기대 결과 |
|---|---|
| GET 요청 | `{"status": "ok"}` 포함 |

### 2-2. /api/parse
| 케이스 | 기대 결과 |
|---|---|
| 정상 카카오톡 .txt 업로드 | speakers, messages, min_date, max_date 반환 |
| 빈 파일 업로드 | 빈 speakers/messages 반환 (에러 아님) |
| 카카오톡 형식이 아닌 파일 | 빈 speakers/messages 반환 |

### 2-3. /api/analyze
| 케이스 | 기대 결과 |
|---|---|
| 정상 요청 (20개 이상) | SSE 스트림 — stage 이벤트 1개 이상, done 이벤트 1개 |
| 메시지 20개 미만 | HTTP 400, detail 메시지 포함 |
| 존재하지 않는 화자 | HTTP 400 (메시지 0개) |
| date_from > date_to | 빈 범위로 400 처리 |

### 2-4. 파싱 단위 테스트
| 케이스 | 기대 결과 |
|---|---|
| 정상 메시지 라인 파싱 | speaker, text, date 정확히 추출 |
| 멀티라인 메시지 | 이전 메시지에 이어붙임 |
| 날짜 구분선 (`---`) | 무시 |
| 이모지 포함 메시지 | 정상 파싱 |

## 3단계 — 프론트엔드 체크리스트 (브라우저 직접 확인)

브라우저에서 http://localhost:5173 에 접속해 직접 확인한다.

### 3-1. 화면 흐름
- [ ] 접속 시 랜딩 페이지가 먼저 표시된다
- [ ] "무료로 시작하기" 클릭 → 파일 업로드 화면으로 전환된다
- [ ] 로고 클릭 → 랜딩 페이지로 돌아간다
- [ ] "← 처음으로" 클릭 → 랜딩 페이지로 돌아간다

### 3-2. 파일 업로드
- [ ] .txt 파일 업로드 시 파싱 후 화자 선택 화면으로 넘어간다
- [ ] 카카오톡 내보내기 가이드가 업로드 아래에 표시된다

### 3-3. 화자/날짜 선택
- [ ] 화자 드롭다운에 파일 내 화자가 모두 표시된다
- [ ] 화자 선택 후 날짜 선택기가 표시된다
- [ ] 메시지 20개 미만이 되는 날짜 선택 시 경고 문구가 표시된다
- [ ] 6000자 초과 범위 선택 시 경고 문구가 표시된다
- [ ] 정상 범위 선택 시 메시지 수/글자 수 안내가 표시된다

### 3-4. 분석 화면
- [ ] 분석 중 MBTI 팁이 4초마다 바뀐다
- [ ] 첫 토큰 도착 후 스트리밍 텍스트가 실시간으로 표시된다
- [ ] 현재 단계("분석 텍스트 구성 중" 등)가 상단에 표시된다
- [ ] 경과 시간이 실시간으로 올라간다

### 3-5. 결과 화면
- [ ] MBTI 4글자 코드가 상단에 크게 표시된다
- [ ] 축별 분석 결과와 헤더 코드가 일치한다
- [ ] "공유하기" — iOS에서 기본 공유 시트가 열린다
- [ ] "다운로드" — PNG 파일이 저장된다
- [ ] "복사하기" — 클립보드에 텍스트가 복사된다
- [ ] "다시 분석하기" — 랜딩 페이지로 돌아간다

### 3-6. 에러 처리
- [ ] 분석 실패 시 JSON 원문이 아닌 한국어 메시지가 표시된다
- [ ] 에러 화면에서 "다시 시작" 클릭 시 랜딩으로 돌아간다

### 3-7. 모바일 레이아웃
- [ ] 브라우저 개발자도구 모바일 뷰에서 레이아웃이 깨지지 않는다
- [ ] 버튼 크기가 터치하기 충분하다

## 4단계 — Playwright E2E 자동화

`frontend/tests/` 에 테스트를 작성하고 실행한다.

### 범위

LLM 호출이 포함된 분석 결과는 비결정적이므로 **LLM 호출 전까지만** Playwright로 자동화한다.
분석 결과 내용 검증은 2단계 pytest에서 백엔드 단독으로 처리한다.

### 4-1. 화면 전환 자동화
| 테스트 | 동작 | 기대 결과 |
|---|---|---|
| 랜딩 진입 | localhost:5173 접속 | "MBTI Analyzer" 텍스트 노출 |
| 시작 버튼 | "무료로 시작하기" 클릭 | 파일 업로드 화면으로 전환 |
| 로고 클릭 | 업로드 화면에서 로고 클릭 | 랜딩 페이지로 복귀 |
| 처음으로 버튼 | "← 처음으로" 클릭 | 랜딩 페이지로 복귀 |

### 4-2. 파일 업로드 자동화
| 테스트 | 동작 | 기대 결과 |
|---|---|---|
| 정상 파일 업로드 | sample.txt 파일 선택 | 화자 선택 화면으로 전환 |
| 화자 드롭다운 확인 | 업로드 완료 후 | 드롭다운에 화자 이름 1개 이상 노출 |

### 4-3. 화자/날짜 선택 자동화
| 테스트 | 동작 | 기대 결과 |
|---|---|---|
| 화자 선택 | 드롭다운에서 화자 선택 | 날짜 선택기 노출 |
| 날짜 경고 — 메시지 부족 | 최대 허용일보다 늦은 날짜 입력 | 경고 문구 노출 |
| 날짜 경고 — 범위 초과 | 최소 허용일보다 이른 날짜 입력 | 경고 문구 노출 |
| 정상 날짜 선택 | 유효 범위 날짜 입력 | 메시지 수/글자 수 안내 노출 |
| 분석 버튼 활성화 | 화자 + 날짜 모두 선택 | "분석하기" 버튼 클릭 가능 |

### 4-4. 에러 처리 자동화
| 테스트 | 동작 | 기대 결과 |
|---|---|---|
| 백엔드 미실행 상태 | 서버 없이 분석 시도 | 에러 화면에 한국어 메시지 노출 |
| 에러 화면 복귀 | "다시 시작" 클릭 | 랜딩 페이지로 복귀 |

### 4-5. 자동화 제외 항목 (사유)
| 항목 | 제외 사유 |
|---|---|
| 분석 결과 내용 | LLM 비결정적 — pytest로 대체 |
| 공유하기 | OS 공유 시트는 브라우저 자동화 불가 |
| 다운로드 파일 내용 | OS 다운로드 폴더 접근 제한 |
| MBTI 팁 전환 | 4초 대기 비효율 — 시각 확인으로 대체 |

### 4-6. 회귀 검증 전략

코드 변경 시마다 아래 순서로 재실행한다.

```
pytest backend/tests/       # 백엔드 API 회귀
playwright test             # 프론트엔드 E2E 회귀
```

핵심 흐름(랜딩 → 업로드 → 화자 선택 → 날짜 선택)이 깨지지 않으면 통과로 간주한다.
